git — распределённая система управления версиями

Статья “Что такое git и зачем он нужен”

Основы Git. Учебник

Интерактивный тренажер по Git

git — c этого слова начинаются все команды

репозиторий git — каталог файловой системы, в котором находятся файлы конфигурации репозитория, файлы журналов, хранящие операции, выполняемые над репозиторием, индекс, описывающий расположение файлов, и хранилище, содержащее собственно файлы

Основные команды

Настройка имени

git config --global user.name "YourName"

YourName — имя

--global — настройка будет применена для всех проектов.
Чтобы настройка применилась только к одному проекту нужно вместо --global указать --local.

Настройка электронной почты

git config --global user.email "workingfish@mail.ru"

workingfish@mail.ru — адрес электронной почты

Проверить настройки

git config --global --list

Вместо --global может быть --local.

Перемещение по папкам

pwd — узнать путь к папке, в которой сейчас находишься

сd название папки — переход в указанную папку

cd .. — переход на одну папку назад

cd / — переход на одну папку вперёд

cd - — вернуться к предыдущей папке

Обзор папки

ls — какие файлы есть в папке

Подготовить к работе репозиторий (делается один раз для одного проекта)

git init

"Init" — это сокращение от "initialize" ( т.е. "инициализироать" или "включать", "запускать").

Это значит, что когда пишем "git init", Git "включается" или "запускается" для данного репозитория (т.е. папки).

Git создает в указанной папке скрытую папку ".git". В этой папке хранятся служебные файлы Git. Кстати, если её удалить, то Git уже не будет "видеть" эту папку (т.е. если мы напишем "git status", нам ответят что это "не Git репозиторий").

Посмотреть состояние репозитория

git status

Красным выделены файлы, которые git не отслеживает

Зелёным выделены файлы, которые git отслеживает

Отслеживать всё

git add .

Отслеживать определённый файл

git add index.html

Просмотр истории коммитов

git log

Зафиксировать изменения

git commit -m "Описание, что изменилось"

Откатить файл до определенного коммита

git checkout c73af1de0877fe25eb4648c2356742f253cc000d index.html

c73af1de0877fe25eb4648c2356742f253cc000d — хэш коммита

Исправить последний коммит

git commit --amend -m "Правильное сообщение коммита"

Создать новую ветку

git checkout -b имя ветки

Перемещение на другую ветку

git checkout имя ветки
